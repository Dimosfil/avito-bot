Драфт A · AI Inbox

Самый близкий к текущей реализации вариант: слева лиды по объявлениям, в центре диалог, справа AI-инсайты и технические детали. Хорош для первого внедрения без сильной перестройки JS.

минимальный риск быстро внедрить фокус на handoff
  • Топбар превращается в статус-центр: Avito, AI, автоответ.
  • Лиды получают явные статусы: AI ведет, нужен менеджер, ручной режим.
  • Правая колонка вместо сырого JSON показывает причину handoff, AI draft, SLA и payload по кнопке.

Драфт B · Handoff Pipeline

Более “продажный” вариант: админка открывается не списком чатов, а воронкой состояний. Подходит, когда менеджеру важнее видеть приоритет и следующую задачу, чем читать все диалоги подряд.

интереснее визуально для руководителя воронка
  • Колонки соответствуют состояниям: новые, AI ведет, нужен менеджер, ручной режим.
  • Карточка показывает источник, объявление, интент, оценку срочности и быстрые действия.
  • Диалог открывается в drawer или отдельной панели по клику на карточку.

Драфт C · Live Desk

Самый современный и “продуктовый” вариант: вертикальная навигация, компактный список событий и большое окно диалога. Хорош, если дальше появятся VK, Telegram, Drom и сайт.

масштабируется мультиканальность современный SaaS
  • Навигация отделяет inbox, pipeline, правила, статистику и настройки каналов.
  • Лента событий объединяет новые сообщения, handoff, ошибки API и AI-статусы.
  • Правая часть показывает диалог и карточку клиента без отдельного API inspector на первом экране.

Быстрые действия

Рекомендация для MVP

Начать с A

Он ближе к текущему DOM: можно модернизировать CSS и переставить правую колонку без большого переписывания логики.

Добавить из B

KPI-строку и состояния воронки как второй экран или режим группировки, когда появится больше лидов.

Держать C как целевую картину

Этот вариант лучше для мультиканальности, но требует больше навигации, роутинга и состояния интерфейса.
